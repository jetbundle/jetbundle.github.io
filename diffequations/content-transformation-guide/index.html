Overview

5. Unified Tone - The ‚ÄúAscent‚Äù Narrative

Principle: Every section must end by explicitly stating the ‚Äúfailure‚Äù or ‚Äúlimitation‚Äù that motivates the next section.

Structure: - End each section with: ‚ÄúThis limitation leads us to [next concept]‚Ä¶‚Äù - Connect to the broader narrative arc - Use transition phrases like ‚ÄúHowever, when‚Ä¶‚Äù, ‚ÄúThe failure of‚Ä¶‚Äù, ‚ÄúThis exposes‚Ä¶‚Äù

Step-by-Step Transformation Process

Step 1: Add Essence Block

Add a high-level quote at the very top, immediately after the title:

# Section X.X: [Title]

> [One-sentence essence that captures the section's core insight]

## Introduction

Step 2: Restructure Mathematical Content

For each concept: 1. Define rigorously (in-line, not assumed) 2. Immediate example (blockquote format) 3. Reflection paragraph (what it revealed/failed)

Step 3: Integrate Examples

Move examples from the bottom to immediately after their relevant theory: - Use blockquote styling: > **Worked Example: [Title]** - Include ‚ÄúThe Nuance‚Äù or ‚ÄúReflection‚Äù paragraph after each

Step 4: Replace Example List with Challenges

Remove the ‚ÄúComplete Examples‚Äù section and replace with: - Two Architectural Challenges - Make solutions collapsible using <details> tags - Ensure challenges synthesize multiple concepts

Step 5: Fix Math Formatting

Critical: Replace all | with \mid EXCEPT in \left| and \right| delimiters.

Find and Replace: - \{x | x > 0\} ‚Üí \{x \mid x > 0\} - P(A | B) ‚Üí P(A \mid B) - Keep: $\left| x \right|$ (absolute value) - Keep: $\left\| \mathbf{v} \right\|$ (norm)

Step 6: Add Narrative Hooks

End each subsection with a transition that: - States a limitation - Connects to the next concept - Maintains the ‚Äúascent‚Äù narrative

Math Formatting Rules

  1. Separators ‚Üí Use \mid
    • Set notation: $\{x \mid x > 0\}$
    • Conditional: $P(A \mid B)$
    • Divides: $a \mid b$
  2. Delimiters ‚Üí Use \left| and \right|
    • Absolute value: $\left| x \right|$
    • Norm: $\left\| \mathbf{v} \right\|$
  3. Never use bare | in math expressions (causes table parsing)

Quality Checks

Before finalizing, verify: - All examples are integrated (none at the bottom) - Two Architectural Challenges replace example list - Solutions are collapsible - Math uses \mid for separators - Narrative flows from concept to concept - Each subsection ends with a hook to the next - Essence block captures the section‚Äôs core
